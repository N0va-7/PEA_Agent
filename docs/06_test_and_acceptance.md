# 06. 测试与验收说明

## 1. 测试目标

测试目标不是只验证接口能返回，而是验证整条邮件安全分析链是否能够稳定运行、复用历史结果并正确展示。

## 2. 建议测试分层

### 2.1 后端单元测试

建议覆盖：

1. URL 提取
2. VT URL 信誉映射
3. URL 风险决策
4. 内容复核 fallback
5. 报告生成

### 2.2 接口测试

建议覆盖：

1. 邮件上传与任务轮询
2. URL 风险接口
3. 静态沙箱任务提交与历史查询
4. 规则管理接口

### 2.3 前端构建与联调测试

建议覆盖：

1. 登录
2. 邮件上传
3. URL 分析
4. 静态沙箱上传
5. 历史详情页展示

## 3. 当前验收重点

当前系统最值得展示的验收点：

1. VT URL 高危可直接触发恶意短路
2. 同一 URL 再次分析会复用 `url_analyses`
3. 同一样本再次提交静态沙箱会复用附件历史结果
4. 历史记录、规则管理和独立 URL 页都能正常工作

## 4. 论文测试报告建议结构

建议写成下面 5 节：

1. 测试环境
2. 功能测试
3. 接口测试
4. 典型样本实验
5. 结果分析

## 5. 适合截图的测试页面

1. 邮件上传任务阶段页
2. 历史记录详情页
3. URL 风险分析页
4. 静态沙箱历史与详情页
5. 规则管理页

## 6. 可直接复用的命令

后端测试：

```bash
./.py311/bin/pytest backend/tests -q
```

前端构建：

```bash
cd frontend
npm run build
```

静态沙箱健康检查：

```bash
curl http://127.0.0.1:8000/healthz
```
