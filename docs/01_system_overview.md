# 01. 系统概述

## 1. 项目定位

PEA Agent 是一个邮件与链接安全分析平台，面向钓鱼邮件、恶意链接和可疑附件的联合检测场景。

系统当前提供 4 条业务入口：

1. 邮件上传分析
2. URL 风险分析
3. 静态沙箱上传扫描
4. 静态沙箱规则管理

## 2. 研究目标

本项目解决的问题不是“单点检测”，而是将多个来源的安全信号组织成一条可解释、可复用、可展示的分析链路。

目标包括：

1. 对邮件正文、链接和附件进行联合分析。
2. 在 VirusTotal 配额受限的前提下，尽量复用历史分析结果。
3. 输出结构化决策结果，而不是只给一个黑盒分数。
4. 提供前端控制台，支持历史查看、复核和规则管理。

## 3. 当前系统能力

### 3.1 邮件主分析链

输入 `.eml` 文件后，系统会完成：

1. 邮件解析
2. URL 提取与归一化
3. VT URL 信誉查询
4. URL 模型评分
5. 正文内容复核
6. 附件静态沙箱分析
7. 综合决策
8. Markdown 报告生成

### 3.2 独立 URL 风险分析

系统支持直接输入 URL 进行分析，无需构造邮件样本。

输出包括：

1. URL 信誉结果
2. URL 模型评分
3. 最终风险结论
4. 历史复用状态

### 3.3 独立静态沙箱

附件静态沙箱是独立服务，支持：

1. 上传单个样本扫描
2. 历史任务回看
3. 风险分与裁决查看
4. YARA 规则列表与编辑

## 4. 技术路线

当前实现采用“工具层 + 工作流编排 + 前端控制台”的结构：

1. 后端工具层负责单个能力执行。
2. 工作流层负责节点编排和结果持久化。
3. 前端负责上传、查看、过滤和解释展示。
4. 静态沙箱作为独立子系统，通过接口接入主平台。

## 5. 论文可强调的创新点

1. 多信号联合检测：URL 信誉、URL 模型、LLM 内容复核、附件静态分析共同参与决策。
2. 结果复用机制：邮件、URL、VT 查询和附件样本都支持缓存复用。
3. 可解释性：输出 `verdict / score / reasons / decision_trace`，便于人工审阅。
4. 工程化落地：前后端分离，支持独立 URL 检查台与静态沙箱控制台。
