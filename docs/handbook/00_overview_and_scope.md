# 00. 总览与范围

## 1. 项目目标

PEA Agent 的目标很直接：
在真实业务里，给邮件做可解释的风险判定，并让人可以持续校正系统。

## 2. 业务闭环（用一句话讲）

`系统先判定 -> 人工再复核 -> 用复核数据手动调参 -> 新参数上线`。

## 3. 系统能做什么

1. 接收 `.eml` 邮件并异步分析。
2. 提取 URL、分析正文、检查附件信誉。
3. 生成最终分数和中文报告。
4. 历史检索、单条删除、全量清空。
5. 人工打标与反馈审计。
6. 融合参数的 Precheck、运行、激活、回滚。

## 4. 系统不做什么

1. 不自动重训基础模型。
2. 不自动在小样本下跑调参。
3. 不把 LLM 输出直接当最终判定分数。

## 5. 关键事实（避免误读）

1. 决策主逻辑在 `analysis.py`，不是在 LLM 节点。
2. 报告节点就算 LLM 失败，也会走固定中文兜底模板。
3. “打标”动作本身只写数据库，不会改模型文件。
4. 生产推荐 MySQL + Redis，开发可回退 SQLite + memory。
