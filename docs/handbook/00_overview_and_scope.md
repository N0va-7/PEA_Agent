# 00. 项目总览与范围

## 1. 项目目标

PEA Agent 旨在构建一套可落地的企业邮件安全分析系统，核心目标：

1. 自动识别钓鱼邮件风险并给出可解释证据。
2. 支持安全运营人员人工复核与反馈闭环。
3. 在不失控的前提下持续优化融合判定参数。

## 2. 业务闭环

系统形成以下闭环：

1. 上传邮件并自动分析。
2. 输出结构化结果与中文报告。
3. 人工标注恶意/正常反馈。
4. 手动触发调参并生成版本。
5. 手动激活版本后作用于后续判定。

## 3. 当前能力边界

### 3.1 In Scope

1. `.eml` 上传、解析、URL/正文/附件多维分析。
2. 融合判定与风险报告生成。
3. 历史查询、单条删除、全量清空。
4. 人工反馈审计与手动调参。
5. 参数版本激活与回滚。

### 3.2 Out of Scope

1. 在线端到端重训正文/URL基础模型。
2. 自动化无门槛调参发布。
3. 全功能 SOAR 编排或跨系统工单流转。

## 4. 关键澄清

1. 项目中“调参”是融合层参数优化，不是 LLM 微调。
2. LLM 主要用于报告内容补全，不直接决定最终恶意分数。
3. 生产建议使用 MySQL + Redis；SQLite 与内存队列是开发兼容路径。

## 5. 当前仓库主线化策略

当前仓库已按主流程精简：

1. 移除历史归档实现（旧版 agent 目录）。
2. 移除未被主流程引用的旧训练副本与演示脚本。
3. 保留运行必需、训练必需、调参必需和文档必需内容。
